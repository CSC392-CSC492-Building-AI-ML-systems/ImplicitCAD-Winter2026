# AI Context

This folder contains curated project materials that are meant to be used as **AI context** (background information) when working with an LLM.

## Purpose

The files in `ai_context/` are **not** part of the core implementation. Instead, they provide structured references that help an AI assistant:

1. Understand the project goals, scope, and constraints  
2. Use consistent terminology and naming  
3. Follow our coding conventions and repository rules  
4. Avoid repeating past mistakes and re solving already made decisions  
5. Produce more accurate, relevant, and maintainable outputs

In short: these documents are here to **improve correctness and reduce hallucinations** by giving the model the same shared context a human teammate would have.

## What belongs here

Typical content includes:

1. Project overview and high level requirements   
2. Data formats, schemas, and key interfaces  
3. Glossary and domain definitions  
4. Prompt templates or workflows we commonly use  

## Maintenance

Treat these files as a living source of truth for AI assisted collaboration:

- Update when requirements or architecture changes  
- Keep it concise and structured  
- Prefer bullet points and clear sections over long narratives
