<language_rules>
ImplicitCAD uses statement-based ExtOpenSCAD syntax — NOT a functional expression language.

VALID syntax examples:
  cube([10, 20, 5], center=true);
  translate([0, 0, 10]) sphere(r=5);
  difference(r=1) { cube([20,20,20], center=true); cylinder(r=5, h=25, center=true); }

SUPPORTED constructs:
  - Primitives:    sphere, cube, cylinder, cone, torus, ellipsoid (3D)
                   square, circle, polygon (2D)
  - Transforms:    translate, rotate, scale, mirror, multmatrix
  - CSG:           union, difference, intersection — all support r=N for rounded blending
  - Extrusions:    linear_extrude, rotate_extrude
  - Control flow:  for, if/else
  - Definitions:   module, function
  - ImplicitCAD extensions: cube(r=N), square(r=N), union(r=N), shell(w=N),
                             torus(r1,r2), ellipsoid(a,b,c), cone(r,h)

UNSUPPORTED — never use these:
  hull(), minkowski(), offset(), resize()
  polyhedron(), surface(), text()
  import(), include <...>, use <...>
  $fa, $fs  (use $fn instead)

IMPORTANT ANGLE NOTE:
  ImplicitCAD trig functions (sin, cos, tan, etc.) use RADIANS, not degrees.
  Use pi for π. Example: sin(pi/2) not sin(90).
  rotate() angles ARE in degrees (the parser converts them automatically).
</language_rules>
